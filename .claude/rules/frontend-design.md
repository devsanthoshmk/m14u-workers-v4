# Frontend Design & Aesthetics

## Design Thinking
Before coding, define a bold aesthetic direction:
- **Purpose**: Identify the core problem solved.
- **Tone**: Commit to a specific flavor (e.g., brutally minimal, retro-futuristic, luxury/refined, playful). Avoid "AI slop" or generic aesthetics.
- **Differentiation**: Focus on making the interface unforgettable.

## Aesthetics Guidelines
- **Typography**: Use beautiful, unique fonts. Pair distinctive display fonts with refined body fonts. Avoid system defaults like Arial or Inter unless intentional.
- **Color & Theme**: Use cohesive palettes via CSS variables. Prefer dominant colors with sharp accents.
- **Motion**: Prioritize high-impact CSS animations and micro-interactions. Use a "staggered reveal" approach for page loads.
- **Spatial Composition**: Experiment with asymmetry, overlap, and generous negative space.
- **Visual Depth**: Use gradient meshes, noise textures, geometric patterns, and layered transparencies to create atmosphere.

## Critical Constraints
- NEVER default to generic AI-generated patterns (purple gradients on white, standard hero sections).
- Match implementation complexity to the vision (maximalist vs. minimalist).
- Ensure animations are smooth and purposeful.
